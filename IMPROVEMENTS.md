# Meetily Improvement Roadmap

Сделано на основе анализа codebase. Приоритизировано по влиянию и сложности.

---

## 🚨 Критические проблемы (исправить срочно)

### 1. Нет лимитов на использование API
**Проблема:** Бесплатные пользователи могут потратить деньги на OpenAI/Groq API без ограничений.

**Решение:** Добавить систему тарифов с использованием.
- Файлы: `database/models.rs` (новая таблица `usage_stats`), `summary/service.rs`
- Сложность: **Medium** (2-3 часа)
- Начать с: free tier = 60 мин/месяц, pro = unlimited

---

### 2. Нет fallback при отказе LLM провайдера
**Проблема:** Если основной провайдер (Groq) недоступен, суммаризация молча падает.

**Решение:** Цепочка провайдеров с auto-fallback.
```rust
// frontend/src-tauri/src/summary/llm_client.rs
// Try Groq → if fail, try OpenAI → if fail, try local Qwen
// Логирование каждого отказа
```
- Сложность: **Hard** (4-5 часов)
- Начать с: обработка errors в `llm_client.rs:180-220`

---

### 3. Блокирующие операции в async контексте
**Проблема:** `block_on()` и `std::thread::spawn` вместо `tokio::spawn` замораживают pipeline.

**Решение:** Заменить на асинхронные операции.
- Файлы: `src/audio/stt.rs`, `src/summary/metadata.rs`, `src/summary/llm_integration.rs`
- Сложность: **Medium** (3 часа на рефакторинг)
- Результат: +30% быстрее обработка больших файлов

---

### 4. Мёртвый код съедает 300+ МБ
**Проблема:** Старые версии audio логики остались в репозитории.

**Решение:** Удалить и заархивировать в отдельную ветку.
- Удалить: `src/lib_old_complex.rs`, `src/audio/*-old.rs`, `src/**/*.backup`
- Сложность: **Easy** (20 минут)
- Экономия: ~300MB в bинаре, проще читать код

## ⚡ Производительность

### 9. Оптимизировать SQL запросы
**Проблема:** Много `SELECT *` вместо конкретных колонок.

**Решение:** Заменить на явные SELECT.
```rust
// Before
sqlx::query!("SELECT * FROM settings WHERE user_id = ?")

// After
sqlx::query!("SELECT id, key, value FROM settings WHERE user_id = ?")
```

**Файлы:** `database/repositories/*.rs` (все SQL)
**Сложность:** **Hard** (4-5 часов, но нудно)
**Результат:** Меньше данных в памяти, быстрее на больших таблицах

---

### 10. Удалить неиспользуемые поля
**Проблема:** Поле `confidence_score` везде везётся, но никто не использует.

**Решение:** Audit и удаление.
- Файлы: `database/models.rs`, `whisper_engine/`, `parakeet_engine/`
- Сложность: **Easy** (1 час)
