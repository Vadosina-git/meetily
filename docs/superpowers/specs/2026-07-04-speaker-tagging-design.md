# Спек: разметка реплик «Я / Не Я» (Path A)

**Дата:** 2026-07-04
**Статус:** утверждён, готов к плану реализации

## Цель

Показывать рядом с каждой репликой транскрипта, кто говорил:
- **«Я»** — голос из микрофона пользователя;
- **«Не Я»** — звук из системного аудио (собеседники в звонке).

Фича опциональная: сверху над диалогом — тумблер показа/скрытия бейджей.

Это подход «Path A» (разделение по физическому источнику звука), а **не**
настоящая пофамильная/поголосовая диаризация. Различает две стороны
(«ты» / «все остальные»), но не отдельных людей внутри звонка.

## Ключевое архитектурное наблюдение

В текущем конвейере (`frontend/src-tauri/src/audio/pipeline.rs`, ~строки
823–877) микрофон и системный звук доступны **раздельно** прямо перед
смешиванием (`mic_window`, `sys_window` из `ring_buffer.extract_window()`).
Сейчас они смешиваются, и на распознавание (VAD + Whisper) уходит один
смешанный поток; поле `source` в `TranscriptUpdate` захардкожено как `"Audio"`.

Структура `TranscriptUpdate` (`audio/transcription/worker.rs`) **уже содержит**
поле `source: String` и время `audio_start_time` / `audio_end_time`. Половина
проводки для фичи уже на месте.

## Компоненты

### 1. Аудио-конвейер (`pipeline.rs`)

- **Файл записи не меняется.** Смешанный трек по-прежнему пишется в WAV
  (STEP 4). Ветку записи не трогаем — это снимает риск сломать саму запись.
- Вместо одного прохода VAD по смешанному звуку — **два раздельных прохода**:
  - VAD + транскрипция по `mic_window` → сегменты с источником `mic` («Я»);
  - VAD + транскрипция по `sys_window` → сегменты с источником `system` («Не Я»).
- Каждый сегмент шлётся в транскрипцию с корректным `source` вместо
  захардкоженного `"Audio"`.
- **Стоимость:** два прохода вместо одного = выше нагрузка на CPU при записи.
  Тишину VAD отсекает, поэтому не буквально ×2, но при активном диалоге ближе
  к ×2. На слабом железе распознавание может отставать. Осознанная плата.

### 2. Хранение (БД + `TranscriptSegment`)

- У каждой реплики сохраняется источник (`mic` / `system`) вместе с уже
  существующими временем начала/конца.
- Совокупность сохранённых `(start, end, source)` образует **карту говорящих**
  (кто-когда говорил). Хранится вместе с транскриптом встречи в БД.
- Требуется добавить колонку/поле источника в схему хранения транскрипта
  (`database/repositories/transcript*.rs`, `TranscriptSegment` в
  `audio/recording_saver.rs`).

### 3. Retranscribe (повторное распознавание)

- Проблема: сохранённый аудиофайл уже смешанный — прямого разделения нет.
- Решение: при retranscribe новый текст **сопоставляется по времени** с ранее
  сохранённой картой говорящих. Каждой новой реплике возвращается тег по
  перекрытию времени с отрезками карты.
- Карта хранится независимо от текста → переживает любое число retranscribe.

### 4. Интерфейс (`frontend/src/app/page.tsx` + компонент диалога)

- Рядом с каждой репликой — бейдж **«Я»** / **«Не Я»**.
- Реплики из двух потоков сортируются по времени начала (`audio_start_time`).
- Сверху над диалогом — **тумблер** показа/скрытия бейджей:
  - глобальный (применяется ко всем встречам);
  - по умолчанию **включён**;
  - состояние запоминается (в настройках).

## Поток данных

```
mic_window  ──► VAD ──► Whisper ──► сегмент(source=mic)     ┐
                                                             ├─► UI (сортировка по времени, бейдж «Я/Не Я»)
sys_window  ──► VAD ──► Whisper ──► сегмент(source=system)  ┘
                                                             └─► БД (текст + время + источник = карта говорящих)

(mixed) ──► WAV-файл записи  [без изменений]

Retranscribe: новый текст × карта говорящих (по времени) ──► теги восстановлены приблизительно
```

## Обработка ошибок и краевые случаи

- ✅ **Разнобой по времени** между двумя потоками — UI сортирует по
  `audio_start_time`.
- ✅ **Тишина в одном потоке** — VAD отсекает, лишней нагрузки нет.
- ✅ **Многократный retranscribe** — карта отдельно от текста, не портится.
- ✅ **Старые встречи без карты** — показываются без тегов, ничего не ломается.
- ⚠️ **Эхо через колонки** — если пользователь слушает через динамики (не
  наушники), микрофон ловит и собеседника; часть «Не Я» может попасть в «Я».
  Программно не устраняется. Фиксируется как ограничение; совет — наушники.
- ⚠️ **Одновременная речь** — отрезки «Я» и «Не Я» пересекаются; на стыке тег
  приблизительный (берётся преобладающий по времени).

## Известные ограничения (явно)

1. Не различает отдельных людей внутри звонка — только «ты / остальные».
2. Retranscribe даёт теги только для встреч, записанных **с включённой фичей**.
3. Точность тегов деградирует при прослушивании через колонки (эхо-просачивание)
   и при одновременной речи.

## Вне рамок (YAGNI)

- Настоящая поголосовая диаризация (pyannote) — отдельная большая задача.
- Переименование «Не Я» в реальное имя участника.
- Разделение нескольких участников внутри системного звука.
